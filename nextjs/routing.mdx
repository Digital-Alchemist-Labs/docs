# 라우팅

Next.js App Router는 파일 시스템 기반의 라우터를 사용합니다.

## 기본 라우팅

`app` 폴더 내의 폴더 구조가 URL 경로가 됩니다:

```
app/
├── page.tsx          → /
├── about/
│   └── page.tsx      → /about
├── blog/
│   ├── page.tsx      → /blog
│   └── [slug]/
│       └── page.tsx  → /blog/:slug
```

## 동적 라우팅

대괄호를 사용하여 동적 세그먼트를 정의합니다:

```tsx
// app/blog/[slug]/page.tsx
export default function BlogPost({ 
  params 
}: { 
  params: { slug: string } 
}) {
  return <h1>Post: {params.slug}</h1>;
}
```

### Catch-all 세그먼트

```
app/docs/[...slug]/page.tsx → /docs/a, /docs/a/b, /docs/a/b/c
```

<Info>
  `[...slug]`는 여러 세그먼트를 배열로 캡처합니다. 예: `/docs/a/b`는 `{ slug: ['a', 'b'] }`가 됩니다.
</Info>

## 라우트 그룹

괄호를 사용하여 URL에 영향을 주지 않고 라우트를 그룹화합니다:

```
app/
├── (marketing)/
│   ├── about/page.tsx    → /about
│   └── contact/page.tsx  → /contact
├── (shop)/
│   ├── products/page.tsx → /products
│   └── cart/page.tsx     → /cart
```

<Tip>
  라우트 그룹은 관련 라우트를 논리적으로 그룹화하거나, 그룹별로 다른 레이아웃을 적용할 때 유용합니다.
</Tip>

## 병렬 라우트

`@` 접두사를 사용하여 같은 레이아웃 내에 여러 페이지를 동시에 렌더링합니다:

```
app/
├── @modal/
│   └── login/page.tsx
├── @sidebar/
│   └── page.tsx
└── layout.tsx
```

## 네비게이션

### Link 컴포넌트

```tsx
import Link from 'next/link';

export default function Navigation() {
  return (
    <nav>
      <Link href="/">Home</Link>
      <Link href="/about">About</Link>
      <Link href="/blog/hello-world">Blog Post</Link>
    </nav>
  );
}
```

### 프로그래매틱 네비게이션

```tsx
'use client';
import { useRouter } from 'next/navigation';

export default function LoginButton() {
  const router = useRouter();
  
  const handleLogin = () => {
    // 로그인 로직
    router.push('/dashboard');
  };
  
  return <button onClick={handleLogin}>Login</button>;
}
```

<Warning>
  `useRouter`는 클라이언트 컴포넌트에서만 사용할 수 있습니다. 파일 상단에 `'use client'`를 추가해야 합니다.
</Warning>

## 라우팅 요약

| 패턴 | 예시 | 설명 |
|------|------|------|
| 정적 라우트 | `/about` | 고정된 경로 |
| 동적 라우트 | `/blog/[slug]` | 단일 동적 세그먼트 |
| Catch-all | `/docs/[...slug]` | 다중 동적 세그먼트 |
| 선택적 Catch-all | `/docs/[[...slug]]` | 선택적 다중 동적 세그먼트 |
| 라우트 그룹 | `/(marketing)/about` | URL에 영향 없이 그룹화 |
