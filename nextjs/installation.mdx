# 설치 및 환경 설정

Next.js 프로젝트를 시작하는 방법을 알아봅니다.

## 시스템 요구사항

<Info>
  Node.js 18.17 이상이 필요합니다. macOS, Windows, Linux를 지원합니다.
</Info>

## 자동 설치

`create-next-app`을 사용하여 새 Next.js 프로젝트를 생성합니다:

```bash
npx create-next-app@latest my-app
```

설치 과정에서 다음 옵션을 선택할 수 있습니다:

```
Would you like to use TypeScript? Yes
Would you like to use ESLint? Yes
Would you like to use Tailwind CSS? Yes
Would you like to use `src/` directory? Yes
Would you like to use App Router? Yes
Would you like to customize the default import alias? No
```

## 수동 설치

직접 패키지를 설치하여 프로젝트를 구성할 수도 있습니다:

<Steps>
  <Step title="패키지 설치">
    ```bash
    npm install next@latest react@latest react-dom@latest
    ```
  </Step>
  <Step title="스크립트 추가">
    `package.json`에 스크립트를 추가합니다:

    ```json
    {
      "scripts": {
        "dev": "next dev",
        "build": "next build",
        "start": "next start",
        "lint": "next lint"
      }
    }
    ```
  </Step>
  <Step title="디렉토리 생성">
    기본 프로젝트 구조를 생성합니다.
  </Step>
</Steps>

## 프로젝트 구조

```
my-app/
├── src/
│   └── app/
│       ├── layout.tsx
│       ├── page.tsx
│       └── globals.css
├── public/
├── next.config.ts
├── package.json
└── tsconfig.json
```

<Accordion title="각 파일의 역할">
  | 파일/폴더 | 설명 |
  |-----------|------|
  | `src/app/` | App Router의 라우트 디렉토리 |
  | `layout.tsx` | 공유 레이아웃 컴포넌트 |
  | `page.tsx` | 페이지 컴포넌트 |
  | `public/` | 정적 파일 디렉토리 |
  | `next.config.ts` | Next.js 설정 파일 |
</Accordion>

## 개발 서버 실행

```bash
npm run dev
```

브라우저에서 `http://localhost:3000`을 열어 확인합니다.

<Check>
  개발 서버가 정상적으로 실행되면 Welcome to Next.js 페이지가 표시됩니다.
</Check>
